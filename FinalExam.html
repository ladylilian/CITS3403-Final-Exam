⚪ Overview

Creating account

  1. Visit the 24 Game on your preferred browser.
  2. You will be directed to the Register page. Enter your provided Username and Password.
  3. Click Register to register an account.

Logging In

  1. Visit the 24 Game on your preferred browser.
  2. You will be directed to the Register page. Enter your provided Username and Password.
  3. Tick the box next to Remember Me for your credentials to be saved for next time.
  4. Click Login to login.

How to play

Players are randomly given four digits.
The objective is to use the four operators of addition(+), subtraction(-), multiplication(×), and division(÷)
to calculate the four given digits and make the result equals to 24.

  1. Click a Number.
  2. Click an Operator.
    Repeat 1. and 2. to make the 4 digits equals to 24.
  3. Once the players make to 24, the Score will add 100 .
  4. When the players used all the 4 digits, but the result does not equal to 24
    The Reset button will spin and hint them to reset the game.
  5. When the players want to give the stage, they can hit the Give Up button to get the answer and restart the game.
    (The current score will be 0.)

Buttons on header

  1. At any stage of the game, you can click the Help button at the top right to help you understand the rules of the game.
  2. Share button at the top right to help you sharing your score.
  3. Rank button at the top right to show your current score and top 10 players' scores.

⚪ Client-Side

Overview of client-side architecture

We create two folders called templet and static to store the HTML files and the CSS & JavaScripts files.

The contents of our application, such as the header, login, and game pages, as well as additional features such as the guide, ranking, and give-up button, are all contained in the templet file.
In the static file, we further created two files, called javascripts and styles to store JavaScripts and CSS files respectively.
The game_page.js enables users to interact with our app, including the clicking the number and operator buttons to do the calculations and pop-up guide and give-up buttons by clicking.
Finally, the styles contains all the CSS files to style our website.

These files are able to be rendered completely in the browser or requested by the client.

③ Server-side rendering vs client-side rendering

There are four mahor factors between these two:

  1. Load time
  2. SEO (Search Engine Optimization)
  3. Time of appearance against time of first user interaction
  4. Complexity of UI

Options Server-side rendering (SSR) Client-side rendering (CSR)
Advatages
  • Shorter page load time.
    The server responds by sending pre-rendered HTML, so the user will see a non-interactive page immediaately (the JavaScripts are downloading).
  • SEO-friendly.
    Google's algorithms always look for the pages offering the best user experience. Shorter loading time helps the site rank higher in Google search results.
  • Ideal for websites that only deliver static content.
  • Less / no JavaScripts dependency.
  • Lower server usage level. Because it doesn't need to deal with extensive HTML, more processing power is available to load the users' subsequent requests, improving the overall experience.
  • Make the navigation agile.
  • Renders faster after initial page load.
  • Rich site interaction.
Disdvatages
  • Larger document size.
    A new HTML file is loaded each time the user navigates to another page on the website.
  • Unable to support many visitors. When the server has a lot of visitors or the site is vast, the page rendering can take a long time.
  • Server costs will be high to provide higher performance.
    As a result of the increased server demand, a more robust hosting solution is required.
  • Slow loading at the first time. It's a one-time occurrence, but it can give the page a negative first impression.
  • Hinders SEO if it's not properly executed. Because it is based on JavaScript and the content is not constructed until it reaches the client's browsers, search engine bots are unable to fully crawl the sites.

⚪ Server-Side

Overview of server-side architecture

We use Flask, Flask-SQLAlchemy, and SQLite to store players' account information (e.g. username and password).
It can authenticate a user to allow them login back by using the matched username and password stored in our database.
In the pop-up ranking, it shows the top 10 players' usernames and personal highest score records in the game by requesting the date form server.
Using @login_required decorator allows the game page is only accessible to logged-in users.
